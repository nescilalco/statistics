Theory/Research - Median

Median

The median of a data set is the value that divides the data into two equal halves when the numbers are arranged in order. If there is an odd number of data points, the median is the middle value. If there is an even number of data points, the median is the average of the two middle values.

Absolute Deviation and Minimization

The absolute deviation is the sum of the absolute differences between each data point \( x_1, x_2, \dots, x_n \) and a central point \( c \):

\[ f(c) = \sum_{i=1}^{n} |x_i - c| \]

To understand why the median minimizes this function, let’s break it down step by step:

Absolute Deviation and Distance

The absolute deviation measures the total "distance" between each point in the dataset and the chosen central value \( c \). The goal is to find the \( c \) that minimizes this total distance. Unlike squared differences (used in variance), absolute differences don’t exaggerate larger deviations, which makes this method less sensitive to outliers.

Behavior of \( f(c) \)

If you were to move \( c \) either left or right from a central position (e.g., the median), the sum of absolute deviations would either increase or decrease depending on the number of points on either side of \( c \).

If \( c \) is to the left of the median, there are more points on the right, and moving \( c \) slightly to the right will reduce the absolute deviations from these points. Similarly, if \( c \) is to the right of the median, there are more points on the left, and moving \( c \) to the left will reduce the absolute deviations from the points on the left.

Thus, the total deviation will be minimized when \( c \) is positioned such that the number of points to the left and right of \( c \) are balanced—this occurs at the median.

The Median as the Minimizer

  • For an odd number of data points, the median is the middle value, so the number of points on either side of \( c \) is equal. Moving \( c \) in either direction from this point will increase the sum of absolute deviations.
  • For an even number of data points, the median is typically defined as the average of the two middle values. Again, this point ensures that the total deviation is minimized because the number of points on either side is balanced.

In other words, the median is the point at which the sum of the distances between each data point and \( c \) is minimized because it splits the dataset into two equal halves, balancing the deviations on both sides.

Visual Intuition

Imagine lining up all data points on a number line. If you place \( c \) at any value other than the median, you’ll notice that the sum of the absolute distances from the points on one side will outweigh the distances on the other. Only at the median are these distances balanced, which minimizes the total absolute deviation.

Conclusion

Thus, the median is the value of \( c \) that minimizes the total absolute deviation, because it divides the data into two equal halves, making the deviations from the central point as small as possible on both sides. This property makes the median a robust and effective measure of central tendency when minimizing total deviation is important.

Formal Proof

Consider the Ordered Data:

\[ x_1 \leq x_2 \leq \dots \leq x_n \]

Define \( m \) as the median of the data set. For simplicity, assume \( n \) is odd, so the median is:

\[ m = x_{\frac{n+1}{2}} \]

Derivative Analysis

To minimize the sum

\[ f(c) = \sum |x_i - c| \]

we examine how the function behaves as \( c \) changes. The absolute value function \( |x_i - c| \) behaves differently depending on whether \( c \) is greater than, less than, or equal to each \( x_i \).

Define the left and right derivatives of \( f(c) \):

For \( c < x_i \), \( |x_i - c|=x_i - c \).

For \( c > x_i \), \( |x_i - c| = c - x_i \).

Summing over all data points, the derivative of \( f(c) \) with respect to \( c \) is:

\[ f'(c) = \sum_{i=1}^{n} \text{sign}(x_i - c) \]

where \( \text{sign}(x_i - c) \) is +1 if \( c < x_i \), -1 if \( c> x_i \), and 0 if \( c = x_i \).

The median

At the median \( m \), there are an equal number of data points on either side:

For \( c = m \), half of the data points are less than or equal to \( m \) and half are greater than or equal to \( m \).

The sum of signs for data points below the median cancels out with the sum of signs for data points above the median:

\[ f'(m) = 0 \]

Conclusion

Therefore, the median is the value of \( c \) where the sum of absolute deviations reaches a minimum. Moving \( c \) to either side of the median increases the sum of deviations because there will be more points farther away, leading to a larger total distance.

Theory/Research - Location

A "location" in statistics is also called "central tendency" and can be seen as a synthesis of a statistical distribution, as it "summarizes" or "condenses" the information provided by a distribution. It's fundamental to reduce the complexity of a dataset, and does it by extracting key characteristics. This type of statistic aims to identify a value that represents some sort of "center" or "summary" of the data.

Basic Concepts

  • Arithmetic Average (mean): the sum of all values divided by the number of values.
  • Median: the value that separates the lower half from the upper half of the ordered data.
  • Mode: the value that appears most frequently in a dataset.

Other Possible Definitions

  • Weighted mean: variation of the simple arithmetic mean, where each data point contributes differently to the final average based on a corresponding weight.

    • The weighted mean \( \bar{x}_w \) of a set of values \( x_1, x_2, \dots, x_n \) with corresponding weights \( w_1, w_2, \dots, w_n \) is given by:

    \[ \bar{x}_w = \frac{\sum_{i=1}^{n} w_i x_i}{\sum_{i=1}^{n} w_i} \]

  • Geometric mean: used for proportional data (like growth rates), as it's more sensitive to small values. It is calculated by multiplying all the numbers in a dataset and then taking the n-th root (where n is the total number of data points).

    The geometric mean \( G \) of a set of values \( x_1, x_2, \dots, x_n \) is given by:

    \[ G = \left( \prod_{i=1}^{n} x_i \right)^{\frac{1}{n}} \]

  • Harmonic mean: applicable for data that represents rates or ratios. It is based on the reciprocals of the values. It gives more weight to smaller values, making it a good measure when the data involves rates like speed, efficiency, or when averaging quantities measured per unit.

    The harmonic mean \( H \) of a set of values \( x_1, x_2, \dots, x_n \) is given by:

    \[ H = \frac{n}{\sum_{i=1}^{n} \frac{1}{x_i}} \]

  • Quadratic mean: measure of the average magnitude of a set of values. It is particularly useful when dealing with quantities that can be both positive and negative, such as electrical signals or vibrations.

    The quadratic mean \( \bar{x}_{\text{quad}} \) of a set of values \( x_1, x_2, \dots, x_n \) is given by:

    \[ \text{RMS} = \sqrt{\frac{1}{n} \sum_{i=1}^{n} x_i^2} \]

  • Centroids or spatial mean: refers to the central point or the "average" position of a set of points in space. Commonly used in geometry, physics, and data analysis to build clusters.

    The centroid \( C \) of a set of points \( (x_1, y_1), (x_2, y_2), \dots, (x_n, y_n) \) is given by:

    \[ C_x = \frac{1}{n} \sum_{i=1}^{n} x_i, \quad C_y = \frac{1}{n} \sum_{i=1}^{n} y_i \]

  • Midrange: the average of the highest and lowest values in a data set. It gives a simple measure of central tendency but can be sensitive to outliers.

    The midrange of a set of values \( x_1, x_2, \dots, x_n \) is given by:

    \[ \text{Midrange} = \frac{x_{\text{max}} + x_{\text{min}}}{2} \]

  • Trimmed Mean: a robust measure of central tendency obtained by removing a fixed percentage of the smallest and largest values before calculating the arithmetic mean. This helps reduce the impact of outliers.

    The trimmed mean of a set of values \( x_1, x_2, \dots, x_n \) is calculated after discarding the lowest \( p\% \) and highest \( p\% \) of the data:

    \[ \text{Trimmed Mean} = \frac{1}{n-2k} \sum_{i=k+1}^{n-k} x_i \]

  • Midhinge: the average of the first and third quartiles (Q1 and Q3) of a data set. It’s a measure of the center of the interquartile range and is less affected by outliers than the midrange.

    The midhinge of a set of values is given by:

    \[ \text{Midhinge} = \frac{Q_1 + Q_3}{2} \]

    • Observations

    The generalization of "location" statistics or central tendency measures can potentially lead to infinite other definitions, because each method of summarizing the center or typical value of a data set relies on specific assumptions, goals, and interpretations of the data.

    Many central tendency measures, like the Weighted mean, rely on applying different weights to the data points. We could define new types of means by varying how the weights are applied, for example by giving more emphasis to recent values in a time series, or on how far they are from some reference point.

    Central tendency measures often differ in how they treat extreme values (outliers), Trimmed mean, for example, ignores a portion of highest and lowest values. We could create new measures by developing custom rules for dealing with outliers, such as progressive trimming, weighting outliers less, or defining outlier boundaries dynamically.

    Central tendency can also be defined based on certain functions of the data, Mode, for example, is based on the frequency of values. We could define new means based on different mathematical functions, by doing this there are theoretically infinite ways to define new measures.

    The idea of central tendency can be changed and expanded in many ways and with different methods, and each method shows a different way of looking at what matters in a set of data. This makes central tendency a flexible concept, allowing for new ideas and definitions based on the data or situation. Because of this, there’s no limit to how we can define a "center" in statistics.

Attack Simulation Exercise

This code implements a simulation of cyber attacks, where the cumulative number of successful penetrations by a certain number of attackers (defined by the user) is tracked over a time interval, tending to infinite and divided into very small subintervals. The graphical output shows the temporal evolution of the attacks as time progresses and visualizes the trajectories of each attacker.

Input Fields and User Interface

The form contains three input fields to take values for time (n), number of attackers (m), and the attack rate (lambda).

  • Time (t) represents the number of time intervals and it should be very big.
  • Number of Attackers (m) representing the number of entities trying to penetrate the system.
  • Rate of Attacks (lambda) defines the probability of success for each small time interval.

Simulation Logic

The SimulatePenetration() function is where the core of the simulation takes place. Here's how it works:

  • Initialization: A Bitmap object is created to visualize the results, and a random number generator is initialized for probabilistic simulations.
  • Time Step Size: The time window is divided into n small intervals, with each interval having a size dt = 1/n.
  • Probability of Attack Success: For each small interval, the probability of a successful attack is lambda_dt = lambda * dt. This reflects the continuous-time approximation requested by the professor.
  • Simulating Attacks: For each attacker, the program simulates whether an attack is successful at each time step. If the random number generated is less than lambda_dt, the attacker is successful, and the cumulative success count is incremented.
